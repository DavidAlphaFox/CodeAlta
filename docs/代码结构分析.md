## 0. 项目概述

CodeAlta 是一个用 **.NET 10** 开发的「智能体式（agentic）AI 编码 CLI 助手」，通过 `alta` 命令进入一个终端工作区（TUI）。它的核心能力：

- **面向键盘的终端工作区**：标签页、提示词编辑器、项目侧边栏、命令发现、模型选择器、上下文状态、可检视的时间线卡片。
- **provider 中立的模型接入**：OpenAI/Azure/Alibaba、Anthropic、GitHub Copilot、xAI Grok、Google Gemini/Vertex、自定义端点统一接入。
- **持久化的会话**：项目级历史以 CodeAlta 自有 JSONL 日志（journal）保存，可独立于 provider 启动恢复，支持排队、转向（steer）、压缩（compact）。
- **可信本地扩展点**：源码插件（`plugin.cs`）、Agent Skills 风格的 skill 文件夹、会话内的 `alta` live 工具。

技术基线：

| 项 | 值 |
| --- | --- |
| 运行时 | .NET 10（`global.json` 锁定 `10.0.100`） |
| 语言风格 | `Nullable` 启用、`ImplicitUsings`、file-scoped namespace、warnings-as-errors |
| TUI 框架 | **XenoAtom.Terminal.UI**（配套 `XenoAtom.Ansi`、`XenoAtom.CommandLine`、`XenoAtom.Logging`、`XenoAtom.Glob`） |
| 解析库 | SharpYaml（项目/会话元数据）、Tomlyn（配置） |
| 测试 | MSTest |
| 打包 | dotnet-releaser（提交前缀有自动 label 约定） |

## 1. 项目架构

### 1.1 解决方案分层（22 个工程）

按依赖方向自底向上分为 5 层。依赖箭头：上层依赖下层，**下层从不反向依赖上层**。

```
┌──────────────────────────────────────────────────────────────┐
│  表现层 (Frontend / TUI)                                       │
│  • CodeAlta              —— alta 可执行程序，终端 UI 与编排接线    │
├──────────────────────────────────────────────────────────────┤
│  编排层 (Orchestration / 会话运行时)                            │
│  • CodeAlta.Orchestration —— SessionRuntimeService、AgentHub、  │
│                              单写者 mailbox actor、CodeAltaHost  │
│  • CodeAlta.LiveTool       —— 会话内 `alta` 命令网关             │
├──────────────────────────────────────────────────────────────┤
│  扩展层 (Plugins / Skills)                                     │
│  • CodeAlta.Plugins.Abstractions —— 公开的插件创作 API          │
│  • CodeAlta.Plugins              —— 插件发现/编译/加载运行时      │
│  • CodeAlta.Plugin.GitHub / .Mcp / .Statistics —— 内置插件       │
├──────────────────────────────────────────────────────────────┤
│  Agent / Provider 层                                          │
│  • CodeAlta.Agent          —— 会话/事件/工具/provider 契约与运行时 │
│  • CodeAlta.Agent.OpenAI / .Anthropic / .Copilot / .Xai /      │
│    .GoogleGenAI            —— 各 provider 协议适配               │
│  • CodeAlta.Agent.ModelsDev.Updater —— 离线模型目录快照更新工具    │
│  • CodeAlta.Acp / .Acp.Generator —— Agent Client Protocol 客户端 │
├──────────────────────────────────────────────────────────────┤
│  目录 / 配置层 (Catalog)                                       │
│  • CodeAlta.Catalog        —— 项目/会话/skill 目录、TOML 配置     │
└──────────────────────────────────────────────────────────────┘
```

测试与工具工程：`CodeAlta.Tests`、`CodeAlta.Orchestration.Tests`、`CodeAlta.Plugins.Tests`、`CodeAlta.Plugins.Abstractions.Tests`、`CodeAlta.Catalog.Tests`、`CodeAlta.Tests.TinyMcpServer`、`PlaygroundShared`。

> 这些边界由 `src/CodeAlta.Tests/ArchitectureGuardrailTests.cs` 以单元测试强制约束（前后端分离、运行时事件流有界、无宽泛 UI 回调、actor 所有权需文档化等）。

### 1.2 核心边界规则

- 可复用的会话编排只能放在 `CodeAlta.Orchestration`，**不能**进入 `src/CodeAlta` 的 View/Dialog。
- `CodeAlta.Orchestration` 是无头（headless）的，仅引用 `CodeAlta.Agent`、`CodeAlta.Catalog`、`CodeAlta.Plugins`。
- View/Dialog 只渲染状态、调用命令/服务接口；**不得**直接调用 `SessionRuntimeService`、`AgentHub`、provider 注册表或插件运行时。
- Model provider 拥有协议适配、凭据、就绪状态、模型元数据；**不**拥有持久化会话列表或项目/会话恢复——后者归 `IAgentSessionStore`/`AgentSessionCatalog`。
- `CodeAlta.Plugins.Abstractions` 是公开的插件创作面；运行时适配器归 `CodeAlta.Plugins` 或 `CodeAlta.Orchestration`。
- 禁止任何静态可变状态（AGENTS.md 硬性规则）；同会话可变状态走单写者 mailbox actor。

### 1.3 启动与组合（composition）链路

```
Program.cs
  └─ CodeAltaOwnedServices.CreateAsync()         进程级关注点
       │   • 默认根目录 ~/.alta、日志、模型目录刷新、全局配置、provider 注册
       └─ CodeAltaHost.CreateAsync()             组合可复用运行时服务
            • ProjectCatalog / SessionViewCatalog / AgentSessionCatalog / SkillCatalog
            • PluginRuntimeManager + 插件资源适配
            • ModelProviderRegistry / IModelProviderInitializationService
            • AgentHub + SessionRuntimeService
            • ProjectFileSearchService（提示词附件/文件选择）
  └─ CodeAltaFrontendComposition.Create()        在共享服务之上组装 TUI
       • view models、ShellStateStore、frontend events、各 Coordinator、alta dispatcher
```

要点：**provider 初始化** 与 **会话目录加载** 是两条独立启动轨——provider 还在探测时，本地会话已可见；列出会话不会实例化或查询任何 provider。无头/工具驱动路径可在不加载终端控件的情况下复用 `CodeAltaHost`。

## 2. 组件关系和功能（按层详解）

### 2.1 目录/配置层 —— `CodeAlta.Catalog`

CodeAlta 全局状态的元数据存储，主目录为 `~/.alta/`。

| 组件 | 功能 |
| --- | --- |
| `ProjectCatalog` | 从 `~/.alta/projects/*.md`（YAML frontmatter）加载/持久化/查询项目描述符；支持按 slug/id/path 查询，`UpsertFromPathAsync` 为新路径自动建描述符 |
| `SessionViewCatalog` | 管理本地会话 UI 状态与 host 自有内部会话元数据（YAML） |
| `AgentSessionCatalog` | provider 无关的持久化会话缓存，带版本失效；负责列出/读历史/删除（单一 sessions root） |
| `SkillCatalog` | 从多个 skill 根（项目/用户/内置/插件资源）发现并读取 `SKILL.md`，含遮蔽（shadowing）规则 |
| `CodeAltaConfigStore` | 加载/解析/校验 TOML 配置，管理 provider 定义与插件开关 |
| `CatalogOptions` | 定义目录布局（projects/machines/sessions/cache/config 路径） |

`~/.alta/` 目录布局：

```
~/.alta/
├── config.toml          用户配置：[chat] 默认 provider、[providers.*] provider 定义、[plugins.*] 开关
├── projects/            项目描述符（*.md，YAML frontmatter）
├── machines/            机器画像（*.yaml）
├── sessions/            agent 会话 JSONL journal
├── cache/               本地机器状态缓存
├── plugins/             用户源码插件（项目级位于 {project}/.alta/plugins/）
├── saved_prompts/       用户保存的提示词草稿
└── alta.lock            单实例锁（存当前进程 PID）
```

### 2.2 Agent / Provider 层 —— `CodeAlta.Agent` + 各 provider 包

#### 核心契约（`CodeAlta.Agent`）

| 契约 | 职责 |
| --- | --- |
| `IModelProviderRegistry` / `ModelProviderRegistry` | provider 发现与运行时查找；按 id 惰性实例化并缓存 `IModelProviderRuntime` |
| `IModelProviderRuntime` / `IAgentModelProviderRuntime` | 已初始化 provider 的 Start/Stop/Probe 生命周期；产出 turn executor、暴露模型目录与 descriptor |
| `IModelProviderTurnExecutor` | 执行单轮对话（user→model→response），含流式回调与工具集成 |
| `IModelProviderInitializationService` | 初始化并探测所有已配置 provider，跟踪就绪状态并发布变更 |
| `IAgentSession` | 单个会话：事件流订阅、send/steer/abort/compact、历史检索 |
| `IAgentSessionStore` / `IAgentSessionCatalog` | 把会话元数据/事件持久化到文件系统，支持恢复与历史查询 |
| `AgentRuntime` | CodeAlta 自有会话运行时：管理 provider 注册、状态路径、为 raw-API agent 创建会话 |
| `AgentSession` | 共享会话实现：编排对话、压缩、工具调用与事件持久化（实现体量很大，是运行时核心） |
| `AgentEvent` | 规范化的多态事件模型：内容增量、活动、会话更新、计划、交互、权限、用户输入请求、错误；JSON 序列化进 journal |
| `AgentToolDefinition` / `AgentToolSpec` | 定义可被模型调用的 host 工具（校验过的名字 + JSON schema + 异步处理回调） |

子目录：

- `Runtime/` —— `AgentRuntime`、`AgentSession`、`ChatClientTurnExecutor`、`FileSystemAgentSessionStore`、turn 契约、会话状态、事件 journal。
- `Runtime/Compaction/` —— 上下文窗口管理：摘要、检查点、token 预算、规范化、分块、压缩。
- `Runtime/Tools/` —— 内置工具基础设施（`AgentBuiltInToolFactory`、工具桥、patch 应用、文件类型识别）。
- `ModelCatalog/` —— 模型发现：`ModelsDevDatabaseJson`、`ModelsDevCatalogService`、`AgentModelMetadataEnricher`。
- `Diffing/` —— 统一 diff 生成。
- `Data/` —— 内嵌 models.dev 快照（离线兜底模型目录）。

#### Provider 适配包（统一模式）

每个 provider 包遵循同一插接模式：`*ModelProviderRuntimeOptions`（配置）→ `*ModelProviderRuntime`（门面，内部委托 `AgentModelProviderRuntime`）→ `*TurnExecutor`（轮次执行）→ 暴露 descriptor / 模型目录 / turn executor 给 `AgentRuntime`。

| Provider | 关键类 | 传输/特点 |
| --- | --- | --- |
| OpenAI | `OpenAIChatModelProviderRuntime` / `OpenAIModelProviderRuntimeFactory` / `OpenAIChatTurnExecutor` | OpenAI Chat API / Codex，含协议 trace、extra-body、请求覆写 |
| Anthropic | `AnthropicModelProviderRuntime` + `AnthropicStreamingCompatibilityChatClient` + `AnthropicAdaptiveThinkingChatClient` | Anthropic Messages API（经 `IChatClient` 桥） |
| Copilot | `CopilotDirectModelProviderRuntime` / `CopilotDirectTurnExecutor` | GitHub Copilot Direct API，自定义登录管理与鉴权头 |
| Xai | `XaiDirectModelProviderRuntime` / `XaiDirectTurnExecutor` | xAI (Grok)，OpenAI 兼容 + 自定义 OAuth |
| GoogleGenAI | `GoogleGenAIModelProviderRuntime` + `ChatClientTurnExecutor` | Google Generative AI / Vertex（条件路由） |

`CodeAlta.Agent.ModelsDev.Updater` 是独立 CLI 工具：从 `https://models.dev/api.json` 拉取模型元数据，写入 `CodeAlta.Agent/Data/models_dev_db.json` 作为离线兜底目录。

#### Agent Client Protocol —— `CodeAlta.Acp` / `CodeAlta.Acp.Generator`

- `CodeAlta.Acp`：与外部 agent 进程通信的 JSON-RPC 客户端（`AcpClient` / `AcpJsonRpcTransport` / `AcpProcess`），协议类型用 STJ source-gen（`AcpJsonSerializerContext`）。
- `CodeAlta.Acp.Generator`：从 JSON Schema 生成 `.gen.cs`（`SchemaWalker` → `CSharpEmitter` / `AcpMethodEmitter`），生成请求/响应记录与 `AcpClient` 方法重载。

### 2.3 编排层 —— `CodeAlta.Orchestration` + `CodeAlta.LiveTool`

#### 会话运行时（`CodeAlta.Orchestration`）

| 组件 | 职责 |
| --- | --- |
| `SessionRuntimeService` | 中枢编排：拥有各会话 coordinator、会话生命周期（create/resume/delete）、可恢复会话投影、运行时事件、prompt 的 send/queue/steer/abort/compact、CodeAlta 自管 skill 激活、journal 中的 legacy session-view 元数据 |
| `AgentHub` | 进程内 agent/会话门面：通过 `IModelProviderRegistry` 解析 provider 运行时来创建/恢复可运行会话，拥有活跃会话句柄与单会话 run 协调；**不**负责列出持久化会话、**不**探测 provider |
| `SessionRuntimeEvent` | 净化后的单会话事件基类，子类含 `SessionAgentEvent` / `SessionHostEvent` / `SessionLifecycleRuntimeEvent` / `SessionCatalogRuntimeEvent` / `SessionQueueRuntimeEvent` |
| `SessionActor` | 单会话 mailbox actor：命令排序、生命周期取消、监督决策（失败命令 / 停止 actor） |
| `SessionActorRegistry` | 按会话 id 管理并发 actor（`GetOrCreate` / `TryGet` / `RemoveAsync`），处置与取消传播 |
| `OrchestrationMailboxActor` | 通用有界 mailbox：基于 `Channel<T>` 的单读多写，含普通槽（信号量限流）与保留槽（高优先级 abort/shutdown），`Ask`/`AskReserved` 支持查询 |

子目录：`Runtime/`（核心协调）、`Runtime/Actors/`（mailbox actor）、`Runtime/Plugins/`（插件编排适配与 hook 派发）、`Runtime/Skills/`（skill 激活）、`Runtime/Prompts/` 与 `Runtime/SystemPrompts/`（模板）、`Hosting/`（`CodeAltaHost` 引导）、`Context/`、`content/`（内嵌指令模板）。

**单写者 actor 模型**（详见 `doc/orchestration-actor-model.md`）：同一会话的命令严格串行化，不同会话 actor 相互独立、可并发。普通命令需等信号量槽，保留命令绕过限流（abort/shutdown）；有界 channel（`FullMode=Wait`、`SingleReader=true`）确定性消费；查询命令经 `TaskCompletionSource<T>` 回复。运行时事件用 drop-oldest 的 `BoundedRuntimeEventStream`，保证慢读者不会造成无界内存增长。围绕运行时 map 的锁必须短，**不得**包裹 provider 调用、工具执行、压缩或 journal 写入。

#### 会话内 `alta` live 工具（`CodeAlta.LiveTool`）

进程内、有界 JSONL 命令面，供会话内 host agent 与插件同步调用（非外部 CLI），用于会话检视、控制、元数据管理。

| 组件 | 职责 |
| --- | --- |
| `AltaCommandDispatcher` | 入口：捕获 stdout/stderr，建 `AltaCommandContext`，调注册表，输出扁平 JSONL transcript |
| `AltaCommandRegistry` | 每次调用构建全新 `CommandApp`，汇集命令策略与节点，解析参数并派发 |
| `AltaCommandContext` | 单次调用上下文：调用方身份、host 服务、stdio、cwd、关联 id、输出上限、取消 token |
| `AltaCommandPolicy` | 策略/审计元数据：命令路径、是否需运行时、是否变更、是否破坏性 |
| `IAltaCommandContributor` | 可插拔命令源接口（`CreateCommandLineNodes` + `GetCommandPolicies`） |
| `BuiltInAltaCommandContributor` | 100+ 核心命令：会话 CRUD、send/steer/abort、排队、项目管理、skill 激活、提醒、models、providers、tools、`ask`（结构化用户输入） |
| `PluginAltaCommandContributor` | 把插件命令工厂适配进命令树，强制 root 去冲突（保留 `version`/`ask`/`session`/`skill`/`provider`/`model`/`plugin`/`tool` 等），隔离插件失败 |

### 2.4 扩展层 —— Plugins / Skills

#### 创作 API（`CodeAlta.Plugins.Abstractions`）

公开的插件创作面。插件继承 `PluginBase` 并覆写贡献/回调方法。贡献点（`PluginPoint`）包括：

| 贡献点 | 用途 |
| --- | --- |
| Startup / CommandLine | 早期运行时初始化 / CLI 参数解析 |
| Command / AltaCommand | UI 命令与快捷键 / 会话内 `alta` 命令 |
| AgentTool | 暴露给模型的工具 |
| SystemPrompt / PromptProcessor / PromptEditor | 静态系统提示部件 / 提交前提示词变换 / 提示编辑器附件 |
| BeforeAgentRun / ToolCall / ToolResult / AgentEvent | run 前注入动态上下文 / 拦截改写工具调用 / 变换工具结果 / 观察规范化事件 |
| SessionEventProjection / Compaction | 瞬态时间线投影 / 压缩 hook |
| Ui / Resource | UI 区域贡献 / 静态资源根（prompts、templates、themes、assets） |

关键类型：`PluginBase`（抽象基类，生命周期回调 `InitializeAsync`/`OnActivatedAsync`/`OnPromptSubmittingAsync`/`OnToolCallAsync`/`OnToolResultAsync`/`OnAgentEventAsync` 等）、`IPluginServices`（host 服务聚合：Logger/Ui/State/Workspace/Sessions/Prompts/Agents/Tasks/Alta）、`PluginRuntimeContext`（附加上下文与作用域 Global/Project）、各类 `PluginContribution*` 记录。

#### 插件运行时（`CodeAlta.Plugins`）

进程内插件运行时：发现、编译、加载、激活、贡献适配。

源码插件位于 `~/.alta/plugins/{package-id}/plugin.cs`（全局）或 `{project}/.alta/plugins/{package-id}/plugin.cs`（项目级），包内可含 `plugin.csproj`、`skills/`、`prompts/`、`templates/`、`themes/`、`assets/`。

| 组件 | 职责 |
| --- | --- |
| `PluginRuntimeManager` | 主编排：发现→编译→加载→激活→管理生命周期 |
| `SourcePluginDiscoveryService` | 扫描插件根、校验 package id、识别 sidecar |
| `PluginBuildService` | 首次或陈旧时编译源码插件 |
| `PluginAssemblyLoading` / `PluginTypeDiscoveryService` | 加载已编译程序集、反射发现 `PluginBase` 子类 |
| `PluginContributionRegistry` / `PluginContributionAdapters` | 聚合各活跃插件贡献并转为 host 可消费形式 |
| `PluginChangeMonitoring` | 监视插件根的文件变更以触发热重载 |

加载阶段：发现 → 构建 → 程序集加载 → 类型发现 → 实例化 → 激活（附加上下文、初始化、startup hook）→ 贡献注册。

#### 内置插件

| 插件 | 功能 |
| --- | --- |
| `CodeAlta.Plugin.GitHub` | GitHub issue 选择器（提示词中输入 `#` 搜索 issue），在可用时暴露 GitHub CLI |
| `CodeAlta.Plugin.Mcp` | Model Context Protocol 服务器集成：从配置自动发现并管理 MCP server |
| `CodeAlta.Plugin.Statistics` | 把每轮/每会话统计作为瞬态时间线卡片投影 |

> 扩展不拥有规范 transcript 的持久化；插件派生的时间线卡片是从规范 agent 事件重放出来的瞬态投影。

### 2.5 表现层 / TUI —— `CodeAlta`（可执行程序）

#### 入口与引导

- `Program.cs`：初始化 `CodeAltaLogging`（XenoAtom.Logging，文件滚动 + 内存 UI 缓冲），在 TUI 启动前先起插件运行时，`CodeAltaSingleInstanceGuard` 单实例守卫（`~/.alta/alta.lock` 存 PID），`CodeAltaCrashReporter` 注册全局/任务异常处理，最终在 `Terminal.RunAsync` 循环（主线程）内异步初始化 TUI。
- `CodeAltaCliOptions.cs`：CLI 标志 `--test`、`--plugin-safe-mode`、`--no-plugins`、`--plugins-status` 等，基于 XenoAtom.CommandLine，并接入插件命令行贡献。

#### App/ 组合与 Coordinator 网络

`App/` 承载 TUI 的业务编排，采用 **Coordinator 模式**：每个关注点有命名 coordinator 负责「状态→事件→UI 更新」接线。核心组合根：`CodeAltaApp`（约 45K，TUI 组合门面/兼容面）、`CodeAltaFrontendComposition`（静态工厂 `Create` 装配整图）、`CodeAltaShellController`（启动目录加载、项目/会话打开、可恢复会话发现、运行时事件入队/排空，经 `IUiDispatcher` 编组）、`CodeAltaOwnedServices`（重服务生命周期）。

按关注点分组的主要 coordinator：

- **会话生命周期**：`SessionCreationCoordinator`、`SessionLoadCoordinator`、`SessionCommandCoordinator`、`SessionHistoryCoordinator`、`SessionProviderSwitchCoordinator`。
- **prompt 处理**：`SessionPromptDispatchCoordinator`、`SessionPromptQueueCoordinator`、`PromptDraftUiCoordinator`、`SessionPromptDraftPersistenceCoordinator`。
- **运行时事件投影**：`SessionRuntimeEventCoordinator`、`SessionRuntimeStateReducer`、`SessionRuntimeTimelineRenderer`、`RuntimeEventPump`、`SessionUsageProjectionController`。
- **provider/model**：`ModelProviderInitializationCoordinator`、`ModelProviderPreferenceCoordinator`、`ProviderFrontendCoordinator`、`ConfiguredModelProviderRegistryBuilder`（约 50K，从 TOML 构建 provider 注册表）。
- **shell/工作区/导航**：`ShellSessionStateCoordinator`、`ShellSelectionCoordinator`、`ShellWorkspaceCoordinator`、`ShellCatalogStateCoordinator`、`SidebarCoordinator`、`NavigatorActionCoordinator`。
- **插件桥**：`PluginHostBridge`（约 27K）、`PluginAgentEventObserver`、`PluginTransientEventProjectionStore`、`PluginShellTabService`、`PluginManagementService`。
- **ask 模式**：`AskModeCoordinator`（代码评审/文件评审/普通提问的结构化输入）。

State/Events/Context 子目录：`ShellStateStore`（不可变的 UI-会话投影快照，负责选择、标签、prompt session）、`FrontendEventPublisher`/`ShellFrontendEvent`（shell 事件）、各 `*Context`（会话选择、标签、命令执行、工作区刷新上下文）。

#### Views / Frontend / Presentation

- `Views/`（65 文件）：约 23 个对话框（`ModelProvidersDialog`、`PromptManagementDialog`、`SkillsManagementDialog`、`PluginManagementDialog`、`ModelCatalogDialog`、`ConfigRecoveryDialog` 等）+ 主视图（`CodeAltaShellView`、`SidebarView`、`PromptComposerView`、`AskFileReviewView`、`QueuedPromptListView`）+ 标签/面板呈现器。全部基于 XenoAtom.Terminal.UI 的 `Drawable`/`Control`。
- `Frontend/Commands/`：`ShellCommand`（不可变命令定义，含可执行/可见/标签委托、帮助分类、放置标志、快捷键）、`ShellCommandRegistry`（命令面板/命令栏/快捷键/帮助/放置的共享来源）、`BuiltinShellCommands`、命令面板 presenter。插件的 `PluginCommandContribution` 被适配进同一注册表。
- `Frontend/Help/`：帮助内容构建。
- `Presentation/`：UI 格式化与交互的业务逻辑——`Chat/`（markdown→终端 markup、响应构建）、`Formatting/`（diff、文件变更摘要、用量、工具调用摘要）、`Timeline/`（时间线/工具调用/文件变更呈现）、`Sidebar/`（树投影构建）、`Tabs/`、`Usage/`、`Editing/`（文件编辑状态、编解码、文件选择）、`Styling/`（主题/调色板）。

#### 其他子目录

- `Models/`：DTO 式不可变状态记录（`SessionState`、`ShellSelection`、`QueuedSessionPrompt`、`ChatTimelineModels` 等）。
- `ProviderDefaults/`：`provider_defaults.toml`（内置 provider 定义）。
- `Threading/`：`IUiDispatcher` / `TerminalUiDispatcher` / `UiDispatch`——把后台工作编组回终端 UI 线程的抽象。

## 3. 端到端：一次提示词的生命周期

```
1. 用户/插件/工具 创建全局或项目会话
2. SessionRuntimeService 解析 model provider、项目根、指令、工具、skill 广告
3. AgentHub 用所选 provider 运行时启动/恢复 CodeAlta 会话
4. 会话经 IAgentSession.SendAsync 收 prompt（或 SteerAsync 转向，若支持）
5. provider/工具事件被规范化为 AgentEvent，再投影为 SessionRuntimeEvent
6. AgentSession 把规范事件 + legacy session-view 头/状态 追加到分片 JSONL journal
7. 前端 RuntimeEventPump 把运行时事件投影到标签、时间线、侧边栏、用量指示器、插件投影
8. 空闲会话可排空一条排队 prompt、在历史可安全重放时切换 provider、或压缩
```

数据流（单轮）：

```
用户输入 → AgentSession.SendAsync/SteerAsync → AgentRuntime 提供对话+turn executor
  → IModelProviderTurnExecutor.ExecuteTurnAsync（provider 专属）
  → provider SDK 调用 → ChatResponseUpdate 流 → AgentTurnDelta 回调
  → AgentTurnResponse（assistant 消息、用量、provider 状态）
  → AgentSession 持久化事件 → FileSystemAgentSessionStore
  → AgentEvent（多态 JSON）写入 journal
```

## 4. 关键设计模式总结

1. **Coordinator 模式**：每个 UI 关注点由命名 coordinator 接线 状态→事件→UI。
2. **延迟异步初始化**：重服务在 `Terminal.RunAsync` 内初始化，保持 UI 在主线程。
3. **单写者 mailbox actor**：同会话可变状态串行化，不同会话并发，事件流有界（drop-oldest）。
4. **不可变状态快照**：`ShellStateStore` 等快照状态，coordinator 计算差异后派发更新。
5. **provider 中立**：统一 provider 工作流；不存在 `IAgentBackend` 概念（已废弃），统一称 model provider / session。
6. **契约/事件驱动**：公开 API 暴露 id、请求/响应记录、快照、句柄、事件；内部 mailbox actor 保持 internal。
7. **可信本地扩展**：源码插件、skill、`alta` live 工具均由 host 显式发现，溯源可见；扩展不拥有规范 transcript。
8. **架构护栏即测试**：`ArchitectureGuardrailTests` 把分层边界固化为单元测试。
